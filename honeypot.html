How to set up HoneyPot and how does it work.

Honeypot can be configured to any computing resource from software and networks to file servers and routers. In the most common cases, it is set up as virtual machines on a physical computer and what this does is to allow the computer to run programs and deploy apps where it resides on. An example of this VMware, Oracle VM VirtualBox or Kali Linux which we used during our ITEC 442 labs activities here are the details:

Some types of honeypots and how they work:

  • Deployment types include:

    • Pure honeypots— These are less sophisticated honeypots and can run easily on any virtual machines. Its primary purpose is to monitor intruders and trap them like prisoners through the links connected to the honeypot network.
    • Low-interaction honeypots— This is more like adware type where it behaves in ways that hackers find a services or system attractive, more like seducing them in other words, to collect data from attackers with their own tools like botnets, malware, and worms.
    • High-interaction honeypots— This deployment honeypot is sophisticated to an enterprise level of production infrastructure. In this type, cybercriminals are not very restricted on the things they can do to ensure attackers are able to dive deeper into their intrusion agenda. But this requires experts to handle as it leaves the honey very vulnerable. Regardless of allowing attackers to have high interaction, these systems are still set up on virtual machines to prevent access to the actual system.

    • Other types of honeypots include:

    • Email traps or spam traps: targets spam messages and source IP to add to auto deny to prevent incoming spam mails from the same source.
    • Decoy database: Helps to detect attacks targeting SQL database and related system frameworks in honeypot.
    • Malware honeypot: A low-interaction honeypot deployment that attracts malware attacks by mimicking software apps and APIs.
    • A spider honeypot: With the growing number of websites, web crawlers are equally growing, especially that malicious bots learn nasty ways, downloads, and indexes content from all over the Internet unlike the actual legit Web Crawlers like Google, Bing, and the rest. Spider honeypots can monitor and detect such and block it, including ad-network crawlers.

Other than what I have learned from the class labs, I spent additional time learning more about honeypots, which made me develop a deeper interest in utilizing the application where needed if applicable.

Other than what I discovered from the lab activities, there are many benefits and disadvantages that I have also learned, which include the following. Generally, the advantages of honeypots outweigh the disadvantages, and it is a great Intrusion Detection System (IDS) tool.

Advantages:

  1. Detect attacks on IoT devices and recommend ways to improve security.
  2. It handles illegal traffic on a network.
  3. Honeypot recognizes patterns like similar IP addresses and sources.
  4. It uses light resources, and most virtual machines used for honeypots are available for free.
  5. Generally, it has a low false rate compared to other intrusion detection systems, which can trigger high false alerts.
  6. Many intrusion detection systems ignore internal threats or don't detect them quickly, unlike honeypots, which are very swift in that regard.
  7. It saves time by preventing hacking of the real computer because it's virtual and a decoy.
  8. It's always up to date with the latest information and resources on threats, attack vectors, exploits, and malware in cyberspace.

Disadvantages:

  1. There are so many means cybercriminals direct their threats, and sometimes it's not directed to honeypots. Therefore, one or IT security shouldn't rely on it 100%.
  2. Intruders may find out that one is using a honeypot and subsequently target another system.
  3. Attackers may also feed honeypots inaccurate information to distract a marked honey so they can create spoofed attacks.
  4. Another downside is that attackers could also use honeypots and weaponize them for their attacks since honeypots are just launchpads.
