IEEE Top 10 vulnerabilities 2021

Introduction: As an aspiring information technology professional, I have delved into the realm of cybersecurity and explored the IEEE Top 10 vulnerabilities of 2021. In this reflective e-portfolio, I will discuss the key vulnerabilities outlined by the IEEE framework and reflect on their significance in shaping my understanding of information security. This exploration has allowed me to gain insights into the evolving threat landscape and the importance of implementing robust security measures to safeguard sensitive information and digital assets.

  1. Remote Code Execution (RCE)
    Remote code execution refers to the ability of an attacker to execute malicious code on a target system from a remote location. This vulnerability can lead to unauthorized access, compromise of sensitive data, and potential control over the entire system. Understanding the techniques and countermeasures associated with RCE has highlighted the importance of secure coding practices and robust input validation mechanisms.
  2. Information Disclosure
    Information disclosure vulnerabilities involve the unintentional exposure of sensitive data to unauthorized parties. This could occur due to misconfigurations, weak access controls, or inadequate encryption measures. Recognizing the impact of information disclosure vulnerabilities has emphasized the significance of implementing strong data protection strategies, including proper encryption, access controls, and data classification.
  3. Denial of Service (DoS) Attacks
    Denial of Service attacks aim to disrupt the availability of a system or network, rendering it inaccessible to legitimate users. Such attacks often exploit vulnerabilities in network protocols or overwhelm systems with a high volume of malicious requests. Understanding the techniques employed in DoS attacks has emphasized the importance of implementing robust network defenses, such as intrusion detection and prevention systems, load balancing, and rate limiting mechanisms.
  4. Privilege Escalation
    Privilege escalation vulnerabilities allow an attacker to elevate their privileges within a system or application, granting them unauthorized access and control over critical resources. These vulnerabilities can stem from weak authentication mechanisms, improper authorization controls, or misconfigurations. Studying privilege escalation vulnerabilities has emphasized the significance of implementing strong access controls, least privilege principles, and regular security audits to detect and mitigate such risks.
  5. Cross-Site Scripting (XSS)
    Cross-Site Scripting vulnerabilities occur when untrusted data is improperly handled by web applications, allowing attackers to inject malicious scripts that are executed by users' browsers. This can lead to the theft of sensitive information, session hijacking, or the delivery of malicious content. Learning about XSS vulnerabilities has underscored the importance of input validation, output encoding, and secure coding practices to mitigate the risk of such attacks.
  6. SQL Injection
    SQL Injection vulnerabilities arise when untrusted user input is improperly handled by database queries, allowing attackers to execute arbitrary SQL commands. Exploiting this vulnerability can lead to unauthorized data access, modification, or deletion. Gaining an understanding of SQL Injection vulnerabilities has emphasized the significance of using prepared statements, parameterized queries, and input validation to prevent such attacks.
  7. Security Misconfigurations
    Security misconfigurations refer to the incorrect or insecure configuration of software, systems, or network devices. These misconfigurations can lead to unintended exposure of sensitive information, unauthorized access, or the exploitation of known vulnerabilities. Recognizing the impact of security misconfigurations has highlighted the need for regular security audits, adherence to security best practices, and robust change management processes.
  8. Cross-Site Request Forgery (CSRF)
    Cross-Site Request Forgery vulnerabilities involve the execution of unauthorized actions on behalf of a user without their knowledge or consent. These attacks exploit the trust established between a user's browser and a targeted website. Understanding CSRF vulnerabilities has emphasized the importance of implementing secure session management, generating and validating unique tokens, and implementing appropriate anti-CSRF measures.
  9. Unvalidated Redirects and Forwards
    Unvalidated redirect and forward vulnerabilities occur when web applications redirect users to untrusted or malicious websites without proper validation. This can lead to phishing attacks, malware infection, or the disclosure of sensitive information. Exploring the risks associated with unvalidated redirects and forwards has emphasized the importance of implementing input validation, secure coding practices, and strict URL validation mechanisms.
  10. Insecure Deserialization
    Insecure deserialization vulnerabilities occur when untrusted or malicious data is improperly deserialized, leading to code execution, privilege escalation, or denial of service. Understanding the risks associated with insecure deserialization has highlighted the need for implementing secure serialization libraries, input validation, and implementing least privilege principles when handling serialized data.

3 tier architecture

Three-tier architecture is a client-server software architecture pattern in which the user interface (presentation), functional process logic ("business rules"), computer data storage and data access are developed and maintained as independent modules, most often on separate platforms. It was developed by John J. Donovan in Open Environment Corporation (OEC), a tools company he founded in Cambridge, Massachusetts.



Wi-fi Alliance

The Wi-Fi Alliance is a non-profit organization that owns the Wi-Fi trademark. Manufacturers may use the trademark to brand products certified for Wi-Fi interoperability. It is based in Austin, Texas.
