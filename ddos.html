Learning about SYN DDS

SYN flooding is a type of Denial of Service (DoS) attack that disrupts the functioning of a network or website by overwhelming it with a flood of SYN requests which results to exhausting its resources making it to stop working or responding. The attack exploits the TCP three-way handshake process, which is used to establish a connection between a client and a server.

During the three-way handshake, the client sends a SYN (synchronize) packet to the server, and the server responds with a SYN-ACK (synchronize-acknowledge) packet. The client then sends an ACK (acknowledge) packet to complete the connection. In a SYN flooding attack, the attacker sends a large number of SYN packets to the target system, but either does not respond to the SYN-ACK packets or sends spoofed source IP addresses.
